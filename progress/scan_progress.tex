\section*{Simulator}

All reported quantized accuracies use the fixed simulator on CIFAR-100 RGGB
with \texttt{raw\_noise=True} (50k train / 10k test).
The dense int4 path uses unsigned I16 first-layer input codes in $[0,65535]$,
integer W4/LUT weights, unsigned A4 hidden activations in $[0,15]$, int32
convolution, and precomputed requantization.  BatchNorm is folded before
simulator execution. Runtime forward does not run \texttt{tanh}, k-means,
nearest-LUT search, or weight quantization.

Current LUT: 15 integer entries,
$[-14,-12,-10,-8,-6,-4,-2,0,2,4,6,8,10,12,14]$ with $m=15$.

\section*{PIM Conv Status}

Implemented an explicit PIM-QAT model path and PIM simulator path without
changing the default dense-QKD path.  The current minimal PIM path keeps the
existing BN student model; GBN/PCN are not enabled in this pass.  The PIM path follows
\texttt{pim\_info/scanpcn\_inference\_spec.tex}: row select over
$[18,18,36,72]$ chunks, keep 48 of 144 rows by default, hybrid stage1.1 keep
72 of 144 rows, 7-bit mid-rise tile ADC, digital BN scale/bias, normalized
hidden DAC activations in $[0,1]$, and unmasked stage1.0 / logits layers.
PIM-QAT eval and PIM simulator are numerically matched on the TU checkpoint
smoke input (\texttt{max\_abs\_diff} $\le 6\times10^{-7}$).
PIM-specific training is the next experiment
and is kept separate from the dense int4 QKD results above.
